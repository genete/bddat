Mockup — Controles de selección desde listado

Usa Tab para navegar entre controles. Los datos son comunes a todos los controles.

Control 0
Texto libre — punto de entrada
  • Campo de texto estándar de Bootstrap 5.
  • Su único rol aquí es ser el primer foco de la página.
  • Al pulsar Tab el foco pasa al Control 1.
Control 1
form-select nativo — Bootstrap 5 puro
  • Control <select> con clase form-select.
  • Sin JavaScript adicional.
  • Delegado totalmente al navegador: teclado, accesibilidad y desplegable nativos.
  • No permite filtrar ni buscar dentro de la lista.
  • La flecha del desplegable es la del sistema operativo.
  • Tab mueve el foco al siguiente control.
  • ↑ / ↓ navegan las opciones.
  • Clic con el ratón abre el desplegable directamente.
  • Espacio / Enter abren el desplegable cuando el control tiene el foco.
  • Inicio de letra salta a la primera opción que empiece por esa letra; pulsaciones repetidas de la misma letra alternan entre todas las ocurrencias.
Control 2
Select nativo — tres mecanismos para volver al estado vacío
  • Flecha ↑ desde la primera opción: el placeholder ya no lleva disabled, por lo que al pulsar ↑ desde "Almería" se puede volver a él.
  • Botón ×: superpuesto dentro del control mediante position: absolute en un wrapper. Limpia con clic. El foco vuelve al select.
  • Escape: mientras el select tiene el foco, devuelve el valor a "" (placeholder).
  • Los tres mecanismos son compatibles. El valor retornado es siempre "".
Control 3
SelectorBusqueda — clase JS reutilizable, estilos heredados del tema
  • Instanciada con new SelectorBusqueda('#id', opciones, config). El HTML lo construye la propia clase.
  • autocomplete="off": ↓ va siempre a la lista de opciones, nunca al historial del navegador.
  • Al enfocar o escribir se abre la lista. Filtro contiene, insensible a mayúsculas.
  • "— Seleccione una provincia —" siempre aparece como primera opción; clicarla limpia la selección.
  • Clic en una opción → la selecciona y cierra la lista.
  • Botón × interior → limpia y devuelve el foco al input.
  • Flecha ▾: botón real (tabindex="-1"), alterna abrir/cerrar. Sin foco por teclado.
  • ↓ / ↑ navegan la lista; Enter confirma; Escape limpia.
  • Clic fuera → cierra sin cambiar la selección actual.
  • CSS solo estructura (posicionamiento). Colores, tipografía y bordes heredados del tema activo.
  • Incluye <input type="hidden" name="..."> para envío en formularios.
  • API pública: .getValue(), .setValue(v, t), .clear().
