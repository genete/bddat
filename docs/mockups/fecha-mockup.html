Mockup — Controles de entrada de fecha

Usa Tab para navegar entre controles y forzar el procesamiento de cada campo.

Control 0
Valor enviado: —
input type="date" — nativo del navegador
  • El navegador renderiza su propio calendario y control de fecha.
  • El valor enviado al servidor es siempre yyyy-mm-dd (ISO 8601), sin importar lo que muestre.
  • En Chrome/Edge (Windows, locale es-ES) muestra el formato dd/mm/aaaa pero hay que hacer clic en día, mes o año por separado.
  • No acepta escritura libre: el cursor salta entre día, mes y año con Tab / /.
  • Icono de calendario nativo a la derecha; no se puede suprimir sin CSS específico.
  • Problema principal: escribir 10/2/25 no funciona como en Access.
Control 1
Valor enviado: —
input type="text" — texto libre sin ayuda
  • El usuario escribe lo que quiere; no hay validación ni formato.
  • El valor enviado al servidor es exactamente lo que el usuario ha escrito.
  • Requiere validación en servidor (y posibles errores de formato).
  • Punto de partida para los controles más avanzados.
Control 2
Valor enviado: —
Máscara automática — inserta / al escribir
  • Mientras el usuario escribe dígitos, el control inserta / automáticamente tras el día y el mes.
  • Escribir 10022025 produce 10/02/2025.
  • Backspace funciona con normalidad (borra carácter a carácter).
  • Limitación: obliga a introducir siempre dos dígitos de día y mes (02, no 2).
  • No valida si la fecha es real (ej: 31/02/2025 no se detecta como error).
  • El valor enviado es la cadena visible; el servidor debe parsear dd/mm/yyyy.
Control 3
Valor enviado (ISO): —
Parser inteligente — normaliza al perder el foco
  • El usuario escribe en formato libre: 10/2/25, 10-2-2025, 10.02.25, 10 2 25.
  • Al salir del campo (Tab o clic fuera), el parser normaliza: 10/2/25 → muestra 10/02/2025, envía 2025-02-10.
  • Año de 2 dígitos: 25 → 2025, 99 → 2099.
  • Si la fecha es inválida (ej: 31/02/2025 o texto no reconocible): borde rojo + campo vacío en el hidden.
  • Si el campo se deja vacío: estado limpio, sin error.
  • El input[type=hidden] interno lleva el valor ISO al formulario.
  • Escape limpia el campo visible y el valor hidden.
Control 4
Valor enviado (ISO): —
EntradaFecha — clase JS reutilizable
  • Instanciada con new EntradaFecha('#id', config). El HTML lo construye la clase.
  • Mismo comportamiento de parser que el Control 3.
  • Botón × interior: limpia el campo y devuelve el foco al input.
  • API pública: getValue(), setValue('2025-02-10'), clear(), enable(), disable().
  • onChange(iso): callback al confirmar una fecha válida o al limpiar ('').
  • El template solo necesita un <div> vacío con un id.
